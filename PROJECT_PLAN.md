# MyRich 项目方案 - 基金资产管理系统

**创建时间**: 2026-03-09
**项目阶段**: MVP（最小可行产品）
**目标用户**: 个人投资者

---

## 一、项目目标

构建一个**个人基金资产管理系统**，帮助用户：
1. **了解资产配比** - 清晰看到各类基金的占比分布
2. **追踪资产趋势** - 基于配比变化看资产增长趋势
3. **计算收益率** - 简单的收益率计算（不考虑分红等复杂情况）

**核心价值**: 相比市面记账应用，支持多种资产类型（基金、股票、加密货币等），而不仅仅是现金。

---

## 二、MVP 核心需求

### 2.1 功能需求

**基金管理**
- 用户可以添加基金（基金代码、份额、购买价格）
- 系统通过 API 自动获取当前净值
- 用户可以查看、编辑、删除基金

**数据更新**
- **手动更新**: 用户点击"刷新"按钮立即更新
- **自动更新**: 后台定时自动更新（支持灵活的更新频率：每天、每小时、每分钟）

**资产可视化**
- **占比展示**: 饼图显示各基金的资产占比
- **收益率计算**: 简单收益率 = (当前价值 - 购买价值) / 购买价值
- **趋势分析**: 基于占比变化看资产增长

### 2.2 非功能需求

- **本地存储**: 所有数据本地 SQLite 存储，隐私安全
- **离线可用**: 无需网络依赖
- **个人使用**: 不考虑多用户、权限管理等

## 三、实现路线

### Phase 1: 基金数据管理（Week 1-2）

**目标**: 实现基金的 CRUD 操作和基础数据展示

**关键模块**:
1. **基金数据模型** (Fund Model)
   - 基金代码、名称、份额、购买价格、当前净值、更新时间
   - 计算字段：当前价值、收益率

2. **基金管理 UI**
   - 基金列表页面
   - 添加基金对话框（输入基金代码、份额、购买价格）
   - 编辑/删除基金功能

3. **本地数据存储**
   - 基金表设计
   - 基金 Repository 实现

### Phase 2: API 对接和自动更新（Week 2-3）

**目标**: 实现基金净值的自动获取和更新

**关键模块**:
1. **国内基金 API 集成**
   - 选择稳定的国内基金 API（天天基金、新浪财经等）
   - 实现净值查询接口

2. **自动更新机制**
   - 后台定时任务（支持分钟级更新）
   - 用户可配置更新频率
   - 手动刷新功能

3. **数据缓存**
   - 记录每次更新的时间戳
   - 避免频繁 API 调用

### Phase 3: 资产可视化和分析（Week 3-4）

**目标**: 实现占比展示和收益率计算

**关键模块**:
1. **占比计算**
   - 计算各基金的当前价值
   - 计算各基金占总资产的比例

2. **可视化展示**
   - 饼图：各基金占比分布
   - 统计卡片：总资产、总收益、平均收益率

3. **收益率分析**
   - 单个基金收益率
   - 总资产收益率
   - 基于历史数据的趋势分析

## 四、技术方案

### 4.1 数据流

```
用户输入基金信息
    ↓
本地 SQLite 存储
    ↓
后台定时任务 / 手动刷新
    ↓
调用国内基金 API 获取净值
    ↓
更新本地数据库
    ↓
UI 重新计算占比和收益率
    ↓
展示饼图和统计信息
```

### 4.2 关键技术选型

| 模块 | 技术方案 | 说明 |
|------|--------|------|
| 数据库 | SQLite | 本地存储，已集成 |
| 状态管理 | Provider | 已有框架 |
| 图表 | Syncfusion Charts | 已有依赖 |
| 定时任务 | Flutter Timer / WorkManager | 后台更新 |
| API 集成 | HTTP 客户端 | 获取基金净值 |

### 4.3 数据库设计

**funds 表**
```
id: 主键
code: 基金代码
name: 基金名称
quantity: 份额
purchase_price: 购买价格
current_price: 当前净值
purchase_date: 购买日期
last_update: 最后更新时间
created_at: 创建时间
updated_at: 更新时间
```

**fund_records 表**（可选，用于历史数据）
```
id: 主键
fund_id: 基金 ID
price: 净值
record_date: 记录日期
created_at: 创建时间
```

## 八、关键决策

1. **API 优先于手工录入** - 用户只需输入一次基金信息，后续自动更新
2. **本地存储优先** - 所有数据本地存储，隐私安全
3. **简单收益率计算** - MVP 阶段不考虑分红、定投等复杂情况
4. **灵活的更新频率** - 支持分钟级更新，满足不同用户需求
5. **个人使用优先** - MVP 不考虑多用户、权限等企业级功能
